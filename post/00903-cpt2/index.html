捉虫（CPT2）

张亮

2018/03/03

基本信息

  • 显示名称：捉虫
  • 英文名称：CPT2
  • 题目编号：00903
  • 参考模板：捉虫-初级，数感
  • 任务描述：测查注意力，特别是长时间注意目标的能力。
  • 测查能力：注意力-持续注意

交互界面

屏幕中央以一次一张的方式快速闪现一系列由四种动物组成的卡片。每张卡片闪现后，用户必须在下一张卡片出现前对目标卡片进行操作，之后程序会对其操作给出相应的反馈。在反馈或刺激间隔时间后，立即呈现下一张卡片。

练习。引导的第一屏不宜采用文字，最好制作一个短暂的示意动画来提示（特别是对于低年级儿童）。引导结束后的练习部分至少做20个刺激，保证正确率在80%以上。

详细说明

  • 刺激：虚拟卡片
    • 刺激配置：关键刺激虫子(A)、小鸟(X)。其他刺激：兔子(B)、青蛙(C)、蜜蜂(D)。
    • 刺激呈现
      • 单个刺激呈现时间由参数（maxSt）控制，默认值为200ms。
      • 刺激间隔时间由参数（isi）控制，默认值为1000ms。
  • 操作
    • 目标卡片：A-X，即在出现小鸟（X）时，需要立即回忆出之前的一张卡片是否是虫子（A），如果是虫子，则这张小鸟卡片就是目标卡片。
    • 操作：在下一张卡片出现前，点击卡片（移动APP用户）或按键盘上的左键（PC端WEB用户）。
    • 判断正误：若用户在目标卡片出现后直到下一张卡片出现时仍未完成操作，或者在不符合要求时完成操作，则记为错误；若用户在符合要求的时候完成操作，则记为正确。
    • 反馈：听觉反馈和视觉反馈都需要。
      • 引导练习：需要对正确和错误给出听觉和视觉反馈。
      • 正式测试：不需要对正确和错误给出听觉和视觉反馈，但是要对完成操作给出听觉和视觉反馈（无论是否错误）。
  • 序列
    • 一般要求
      • 相同的刺激必须至少间隔两次才能出现。
    • 特殊要求
      • 本题目需要用户判断呈现的刺激序列中是否出现A-X配对，因此在序列中有一个很重要的设置是A-X出现的比例。同时，也要考虑到A-(非X)、(非A)-X这两种序列的比例。需要保证这三种序列比例一致。
      • 保证每个刺激出现的频率相等，即都为1/5。
  • 序列配置
    • 见序列文件。
    • 数组中每一个元素都是一个json对象，包含三个名称/值(name/value)配对，分别是
      • length：表示序列的长度
      • stim：包含了由刺激名称组成的多个序列
      • scat：包含了由刺激类别组成的多个序列

配置参数

表1: 配置参数总表
参数类别 参数名称 类型 默认值 说明
刺激呈现 maxSt int 200 单个刺激最大展示时间，单位毫秒
刺激呈现 isi int 1000 刺激间隔时间，单位毫秒
刺激序列 endMethod int 2 题目的结束方式（1=按时间结束，2=按刺激个数结束）
刺激序列 duration int 30000 题目整体持续时间（endMethod=1时生效，单位毫秒）
刺激序列 quantity int 100 刺激总个数（endMethod=2时需全部做完）
操作反馈 feedbackCorrectError bool 0 控制正式测评时候是（1）否（0）需要反馈正确与错误
操作反馈 feedbackNoResponse bool 0 控制正式测评时候是（1）否（0）需要提示用户未操作
操作反馈 feedbackSound bool 1 控制正式测评时候是（1）否（0）给出声音反馈
操作反馈 feedbackVisual bool 1 控制正式测评时候是（1）否（0）给出视觉反馈

数据记录

  • 题目结果显示
    • 显示正确率和平均正确反应时间
    • 展示标准分数和超越百分比
  • 提交原始数据如下表（参考标准中关于格式的说明）
表2: 原始数据变量列表
名称 类型 说明
STIM char 刺激名称(STIMulus)，即A、B、C、D或X
SCat char 刺激类型(Stimulus Category)，包括五种类型：‘Target’,‘AnotX’,‘Aonly’,‘Xonly’或’Random’
Resp int 用户是否操作(Response)，0=未操作，1=操作。
ACC int 正确或错误(ACCuracy)，0=错误，1=正确。
RT int 用户完成操作所需时间（单位：毫秒）(Response Time)，若未完成操作，则记为0

试题算分

  • 算分码：101
  • 传入参数：
参数名称 类型 说明
hitRate float 用户在对所有需要操作的刺激中正确操作的比例
FARate float 用户在对所有不需要操作的刺激中错误操作的比例
