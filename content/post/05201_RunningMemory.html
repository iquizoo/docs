基本信息

  • 显示名称：记忆尾巴
  • 英文名称：RunningMemory
  • 题目编号：05201
  • 任务描述：测查自控力，特别是更新记忆内容的能力。
  • 测查能力：自控力-信息更新
  • 参考模板：倒背数，顺背数

交互界面

屏幕中央以一次一个的方式呈现一系列数字，用户需要不断更新记忆以记住序列中最后呈现的四个数字。待所有数字呈现完毕后，屏幕上方显示出与呈现数字个数相同的方框，但仅有最后四个显示为可填入数字的样式（如，将前面数个的样式改为灰色框）。用户此时使用鼠标点击（PC）或触摸（Android/iOS）屏幕下方呈现的数字键盘以输入所记住的最后四个数字。每输入一个，上方那四个可填入数字的方框的相应位置就显示出所输入的数字。直到四个数字输入完毕后，屏幕显示一定时间（试次间隔时间）的空白，之后进入下一个试次。

练习。练习时候需要让用户了解到作答时候顺序的重要性，即必须按原来的顺序点击最后出现的四个数字。

详细说明

  • 刺激
    • 每个试次中会呈现一系列数字，单个数字呈现时间由参数maxSt（默认300ms）控制，数字呈现时间间隔由参数isi（默认1000ms）控制。
  • 序列
    • 本题目中单个试次要呈现的数字个数有三种，即5，7，9个。
    • 正式测评总计12个试次，即每种4个试次。
    • 练习总计3次，即每种1次。
  • 序列配置（参考标准中关于格式的说明）
    • 本题目只有一种序列，因此传输序列的json字符串仅包含一个由两个名称/值配对组成的对象。
      • prac: 练习阶段的3个序列。
      • test: 正式测试阶段的12个序列。

配置参数

表1: 配置参数总表
参数类别 参数名称 类型 默认值 说明
刺激呈现 maxSt int 300 单个数字最大展示时间，单位毫秒
刺激呈现 isi int 1000 数字呈现之间的间隔时间，单位毫秒
刺激呈现 iti int 1000 试次间隔时间，即用户操作完成到下一个试次开始之间的间隔时间，单位毫秒
刺激序列 endMethod int 2 题目的结束方式（1=按时间结束，2=按刺激个数结束）
刺激序列 duration int 30000 题目整体持续时间（endMethod=1时生效，单位毫秒）
操作反馈 feedbackCorrectError bool 0 控制正式测评时候是（1）否（0）需要反馈正确与错误
操作反馈 feedbackNoResponse bool 1 本题目将会一直等待用户操作，但是用户操作完成时需要给一个提示。
操作反馈 feedbackSound bool 1 控制正式测评时候是（1）否（0）给出声音反馈
操作反馈 feedbackVisual bool 1 控制正式测评时候是（1）否（0）给出视觉反馈
