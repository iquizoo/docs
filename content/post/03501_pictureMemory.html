基本信息

  • 显示名称：图片记忆
  • 英文名称：pictureMemory
  • 题目编号：03501
  • 任务描述：测查记忆力，特别是个体记住各种图片信息的能力。
  • 测查能力：记忆力-语义记忆

交互界面

本题目分为学习和测试两个连续的阶段。

学习阶段。本阶段要求用户对（注意不要告诉用户记忆图片的要求）一系列图片刺激（每张图片都是一件日常物体）做判断。判断任务是确定自己是否触摸过该物体。

  1. 图片呈现在屏幕中央，同时下方呈现两个按钮，左侧为是，右侧为否。图片呈现时间为2秒。用户操作后，按钮消失，但图片不消失。
  2. 图片消失后，屏幕保持0.5秒长的空白。之后进入下一个试次。

测试阶段。本阶段要求用户判断自己是否在学习阶段见过屏幕上呈现的图片刺激。判断的选项有旧的，相似和新的三种。

  1. 图片呈现在屏幕中央，同时下方呈现三个按钮，按钮上的文字从左到右依次为旧的，相似和新的。图片最长呈现时间为2秒。用户操作后，按钮和图片均消失。
  2. 图片消失后，屏幕保持0.5秒长的空白。之后进入下一个试次。

引导。本题目在学习和测试阶段分别有引导。

学习阶段的引导和练习。保证用户熟悉学习阶段的流程和操作方法。由于此处的判断没有正误之分，所以练习3次就开始正式的学习阶段。

测试阶段的引导和练习。保证用户理解旧的，相似和新的三个选择的意义，可以使用样例图片（非学习阶段学习过的图片）分别说明。

详细说明

刺激

本题目的所有用作刺激的图片都是日常生活中常见的物体。图片都有相似的两个版本（分别记为a和b）。图片材料的名称由刺激编号后跟上图片的版本构成，如1a.jpg。每一编号对应图片的两个版本的相似程度由1-5表示，数字越小则越相似。

刺激的呈现。本题目分为学习和测试两个阶段。

在学习阶段，每个试次由两部分组成。

  1. 刺激呈现。时间配置参数：最大呈现时间（maxStLrn），默认为2秒。注意用户操作后刺激也不消失。
  2. 空白屏幕。时间配置参数：试次间隔时间（itiLrn），默认0.5秒。

在测试阶段，每个试次也由两部分组成。

  1. 刺激呈现。时间配置参数：最大呈现时间（maxStTst），默认为2秒。注意用户操作后刺激将消失。
  2. 空白屏幕。时间配置参数：试次间隔时间（itiTst），默认0.5秒。

操作

根据不同的设备状态，操作有所不同：

  • 键鼠操作。学习阶段：要求用户必须用键盘上的左右键反应，而不能使用鼠标点击屏幕下方的按钮。键盘的左右键分别对应屏幕上的左右按钮。测试阶段：使用鼠标点击屏幕上的三个按钮进行操作。
  • 触控屏操作。触碰屏幕下方对应的按钮即可。

序列配置

序列文件中各个元素的意义：

  • phase：阶段。即此序列属于学习阶段（Learn）还是测试阶段（Test）。
  • id：图片的编号。请注意本题目图片的命名方式是三位数编号加上字母a或者b（a，b的意义即在刺激一节里提到的单个图片的不同版本）。所有不足三位数的编号均在前面加了相应数目的0，如编号为1则命名时采用001。id和ver共同决定了实际应该选择哪张图片。
  • ver：图片的版本。即刺激一节里提到的单个图片的不同版本。每一张图片命名中已包含了此信息。id和ver共同决定了实际应该选择哪张图片。
  • sim：图片两版本的相似程度。刺激一节里已提及使用1-5刻画同一编号的两个不同版本图片的相似程度，此元素记录了这个数字。
  • type：测试阶段专有。该图片出现在测试阶段时的类型。target表示该图片在学习阶段出现过，lure表示该图片的另一个类似的版本在学习阶段出现过，foil表示该图片是全新的图片。
  • cresp：测试阶段专有。正确答案。old（和target对应）表示正确答案是旧的，similar（和lure对应）表示正确答案是相似，new（和foil对应）表示正确答案是新的。

表2: 原始数据变量列表
名称 类型 说明
Phase char 阶段。Learn或Test，和序列中一致。
id int 刺激编号，和序列中一致。
Ver char 刺激版本，包括2种可能：‘a’、‘b’，和序列中一致。
Sim int 当前刺激两个版本的相似程度，可能取值1-5，和序列中一致。
Type char 图片的类型。学习（Learn）阶段为空文本。测试阶段和序列一致。
CResp char 正确答案。学习（Learn）阶段为空文本。测试阶段和序列一致。
Resp char 用户的实际反应（Response）。学习阶段有三种可能：‘Left’、‘Right’、‘None’（未操作），测试阶段有四种种可能：‘Old’、‘Similar’、‘New’、‘None’（未操作）。
ACC int 正确或错误（ACCuracy）。学习阶段为空文本。测试阶段：0=错误，1=正确，-1=未完成操作。
RT int 用户完成操作所需时间（单位：毫秒）(Response Time)，若未完成操作，则记为0。
