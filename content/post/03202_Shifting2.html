基本信息

  • 显示名称：反应转换
  • 英文名称：Shifting2
  • 题目编号：03202
  • 任务描述：测查思维力，特别是灵活切换思维方式的能力。
  • 测查能力：思维力-思维灵活性
  • 相关任务：思维转换。本题目适用于5年级（10岁）以上儿童，低年级请使用思维转换任务。

交互界面

每一个试次中，屏幕上方呈现一个线索符号（\(\heartsuit\)/<），一定时间（线索目标间隔）后屏幕中央呈现一个目标词语（中文具象名词），线索符号和目标词语会同时停留在屏幕上，直到用户需根据线索对目标做出操作（或达到最大展示时间）后一起消失。最后，屏幕保持一定时间（反应线索间隔）的空白后进入下一个试次。

练习。本题目的线索符号有两种，意味着实际上此题目包含了两个子任务，且每一个试次会要求用户根据程序给定的一个子任务进行反应。因此，练习由三部分组成（详细参看详细说明部分）：

  1. 指引和练习第1个子任务，并保证用户理解第1个子任务。
  2. 指引和练习第2个子任务，并保证用户理解第2个子任务。
  3. 指引和练习在两个子任务混合起来（实际测试时候的形式）的时候如何操作。

详细说明

  • 刺激
    • 线索符号两个：其意义在告知用户当前试次需要执行的子任务。
      • 其一为\(\heartsuit\)，此线索符号要求用户判断目标词语所代表的物体是否有生命(Life)。
      • 另一为<，此线索符号要求用户判断目标词语所代表的物体是否能放入鞋盒(Size)。
    • 目标词语（见下表）
    • 刺激呈现
      • 线索符号先于目标词语呈现，间隔时间为线索目标间隔，由配置参数cti（默认为500ms）控制
      • 目标词语呈现后线索符号不消失
      • 目标词语的最大展示时间由配置参数maxSt（默认为3000ms）控制
      • 用户对目标词语做出操作反应或达到最大呈现时间后，屏幕需在下一个试次开始前（即下一张背景图片出现前）变成空屏状态，持续时间为反应线索间隔，由配置参数rci（默认为500ms）控制。
表1: 目标词语及其属性一览表
编号 名称 大小 生命
1 桌子 大 无
2 飞机 大 无
3 大象 大 有
4 狮子 大 有
5 铅笔 小 无
6 杯子 小 无
7 蚂蚁 小 有
8 蝴蝶 小 有
  • 操作
    • Android/iOS：触碰点击页面下方的按钮（左侧标明“否”，右侧标明“是”）做出对应的反应。例如，在\(\heartsuit\)线索下，如出现词语“蚂蚁”，用户需点击“是”（即“有生命”）。
    • PC/Web：若PC拥有触摸屏，则用户可以使用Android/iOS的操作方式；若PC没有触摸屏，则要求用户使用标准输入设备键盘上的←和→进行操作，分别表示左侧和右侧按钮。
  • 指引和练习
    • 指引和练习分为3个部分：
      1. 指引和练习是否有生命子任务。线索符号依然呈现。至少完成8次，即至少每种目标词语做1次；8次后若正确率不高于80%，则继续练习至正确率高于80%或总计练习16次时结束。注意每个8次练习中都是将所有刺激按随机顺序呈现的（后面的练习也是如此）。
      2. 指引和练习是否能放进鞋盒子任务。线索符号依然呈现。至少完成8次，即至少每种目标词语做1次；8次后若正确率不高于80%，则继续练习至正确率高于80%或总计练习16次时结束。
      3. 指引和练习在两个子任务混合条件。提醒用户注意线索符号确定操作方式。此部分至少包含16个试次，保证所有刺激在两种子任务情况都分别练习1次，采用完全随机的顺序即可。16次后若正确率不高于80%，则继续练习至正确率高于80%或总计练习32次时结束。注意每个16次练习中都是将所有刺激在两种子任务情况都分别随机顺序地练习1次。
  • 序列
    • 一般要求
      • 相同的刺激必须至少间隔两次才能出现。
    • 特殊要求
      • 切换和重复的条件最多连续3次，且连续出现次数1, 2, 3的比例为1:2:1。注意：当重复条件出现\(n\)次时，会重复执行一个任务\(n + 1\)次。
      • 正确操作的一侧最多连续3次，且连续出现次数1, 2, 3的比例为1:2:1。
      • 保证任务类型、切换条件、正确操作相互均衡。
      • 保证刺激在不同的任务类型中比例一致。
  • 序列配置
    • 见序列文件。
    • 注意：本题目的第一个试次是一个“填充”试次，因此若设试次数目为\(n\)时，实际需要完成的试次数目为\(n + 1\)。在序列的配置标准中也是如此，即length为\(n\)序列中，stim个数都是\(n + 1\)。
    • 数组中每一个元素都是一个json对象，包含六个名称/值(name/value)配对，分别是
      • length：表示序列的长度
      • stim：包含了由刺激编号（和“动物图片及其属性一览表”一致）组成的多个序列
      • task：包含了由子任务名称组成的多个序列，其中Life表示判断是否有生命，Size表示判断是否能放入鞋盒
      • cond：包含了由任务条件，即任务是否切换，组成的多个序列，且每个序列中，第一个注明为Filler，其余为Repeat和Switch中的一个
      • cresp：包含了由正确答案组成的多个序列，可能值为Left和Right

表3: 原始数据变量列表
名称 类型 说明
Round int 本任务有轮次的概念，此处记录当前在第几个轮次
STIM int 刺激编号(STIMulus)，1-8中的一个
Task char 子任务，包括2种可能：‘Size’, ‘Life’
Cond char 试次条件(Condition)，即子任务是否切换，包含三种可能：‘Filler’, ‘Repeat’, ‘Switch’

RT int 用户完成操作所需时间（单位：毫秒）(Response Time)，若未完成操作，则记为0
