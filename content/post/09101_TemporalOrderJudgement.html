基本信息

  • 显示名称：顺序判断
  • 英文名称：TemporalOrderJudgement
  • 题目编号：09101
  • 任务描述：测查多感觉整合能力，特别是视听整合能力。
  • 测查能力：感知觉-多感觉整合

前端交互

基本逻辑

屏幕一左一右呈现两个按钮，分别在下面标记“喇叭图片”和“眼睛图片”，任务为间隔一定时间（初始为350ms）分别呈现声音刺激（“哔”）10ms和视觉刺激（颜色上跟背景可区分的圆盘）12ms，两者顺序随机，用户需要根据两者的先后顺序点击相应按钮，记录用户的正确率和反应时。

详细说明

流程如下：

  1. 任务开始。随机先后间隔350ms呈现两种刺激（声音刺激为“哔”，10ms；视觉刺激为颜色上跟背景可区分的圆盘，12ms），让用户根据两者的先后顺序点击相应按钮。当用户正确反应后提高难度，表现为每次两种刺激间隔时间缩短10ms；当用户失败时降低难度，表现为每次增长刺激间隔时间10ms。当用户累计完成8个轮次时，任务结束。
  2. 操作反馈。反馈始终呈现直到用户做出操作，并进入下一试次。
  3. 试次间间隔。给予反馈1s后开始下一个试次。

后台配置

数据记录

表1: 原始数据变量列表
名称 类型 说明
Trial int 试次编号
Round int 轮次编号
Order char 声音（A）和视觉（V）的呈现顺序，包含两种可能：‘AV’，‘VA’
Interval int 两种刺激呈现的时间间隔（单位：毫秒）
Resp char 用户的实际反应(Response)，包含两种可能：‘AV’，‘VA’
ACC int 正确或错误(ACCuracy)，0=错误，1=正确
RT int 用户完成操作所需时间（单位：毫秒）(Response Time)
